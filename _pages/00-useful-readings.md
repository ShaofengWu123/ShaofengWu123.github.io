> What is the difference between serverless and microservices?

**Microservices is a way to design an application, while serverless is a way to run an application (or a part of an application).**

Microservices is a way of designing applications. It describes the paradigm that applications are not managed in a monolithic manner, with a whole piece of code residing on a single server. Instead, a set of loosely connected microservices form a complete application. These microservices can be long-term stateful with their own data(and vice-versa) and they can reside on a physical server that requires developer's management, or just be provided on demand by service providers. 

Serverless is a way of actually constructing the applications. Serverless threads are provisioned on demand statelessly, with no management work for developers. Microservices can be either achieved in a serverless manner or others.  And the code that serverless threads execute can either be a microservice or a whole application.
